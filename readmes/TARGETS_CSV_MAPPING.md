# Inventory Targets CSV to Schema Mapping

## Column Alignment

| CSV Column | TargetUpdate Field | Status | Notes |
|---|---|---|---|
| product_name | N/A (used for commodity lookup) | ✓ | Maps to commodity_id via Commodity.commodity_name |
| product_name_alias | N/A (fallback for matching) | ✓ | Used in fuzzy matching if exact match fails |
| desired_stock_leve | desired_stock_level | ✓ FIXED | CSV has typo (missing 'l'), loader fixes it |
| min_stock_level | min_stock_level | ✓ | Direct match |
| max_stock_level | max_stock_level | ✓ | Direct match |
| target_inventory_holding_days | target_inventory_days | ✓ | Renamed by loader |
| target_cash_realisation_days | estimated_days_to_sale | ✓ | Renamed by loader |
| expected_gross_margin | expected_gross_margin | ✓ | Direct match |
| annual_cost_of_capital | annual_cost_of_capital_rate | ✓ | Renamed by loader (adds "_rate" suffix) |
| monthly_sales_target | (not in schema) | ⊘ | Extra column, dropped by loader |
| target_storage_cap_days | (not in schema) | ⊘ | Extra column, dropped by loader |
| Unnamed columns | (corrupted) | ⊘ | Removed by loader |

## Missing Fields in CSV

These TargetUpdate fields are **not** in the CSV and will use defaults:

| Field | Default | Notes |
|---|---|---|
| cash_realization_rate | NULL | Could be added to CSV if needed |
| is_finalized | False | Set by loader, can be updated via API |
| notes | NULL | Can be added via API after load |

## Data Type Conversions

All numeric fields are converted using `_to_float()` function which:
- Handles percentages (e.g., "12%" → 0.12)
- Strips whitespace and converts strings to float
- Returns None for missing/invalid values
- Handles NaN and None gracefully

## Commodity Matching Strategy

The loader finds commodities using this priority:
1. **Exact match** on `Commodity.commodity_name` (case-insensitive substring)
2. **Alias match** on `product_name_alias` if exact match fails
3. **Fuzzy match** on commodity names with >70% similarity (using SequenceMatcher)
4. **Skip row** if no match found within 70% threshold

This ensures robust matching across different naming conventions.

## Config Date Versioning

- Default: `config_date = today()`
- Can specify custom date via parameter
- If config already exists for same commodity+date → update values
- If new date → create new version (enables history tracking)

## CSV Issues Found & Fixed

1. **Encoding**: File uses latin-1 encoding (Excel export), fixed with `encoding='latin-1'` parameter
2. **Typo in column**: "desired_stock_leve" missing 'l', renamed in loader to "desired_stock_level"
3. **Corrupted columns**: Last 3 columns contain garbage/text that looks like Python code, removed by loader
4. **Percentage values**: expected_gross_margin and annual_cost_of_capital are percentages ("12%"), parsed as decimals (0.12)

## Usage Example
